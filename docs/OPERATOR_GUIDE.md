# Hala Job Operator Guide

This guide is for a non-technical operator managing the Syria-first launch.

## Publish Resources

1. Open admin resource library.
2. Create or update the resource with audience, category, tags, and language.
3. Publish only after checking links and Arabic/English text.
4. Use analytics to see saves, starts, and completions.

## Approve Company Public Profiles

1. Open admin public profile review.
2. Check company name, public description, benefits, logo/cover, hiring process, and links.
3. Approve clean profiles.
4. Reject with a clear note if content is incomplete, misleading, or unsafe.

## Manage Communication Templates And Logs

1. Open communication logs.
2. Filter by channel, status, event, user, or company.
3. Failed email/SMS should be checked against provider configuration.
4. Manual WhatsApp links are copy/share helpers, not official auto-sends.

## Manage CV And Message Templates

1. Review CV templates before featuring them in CV Studio.
2. Keep template preview images, Arabic labels, and English labels aligned.
3. Update message templates through the communication template tools.
4. Test a template before using it for a launch-wide notification.

## Rebuild Salary Insights

1. Open admin salary insights.
2. Run rebuild after large job imports or salary-data corrections.
3. Verify confidence and currency before using salary guidance in launch reporting.

## Manage Campus Partnerships

1. University reviews company partner requests.
2. Approve only real partner companies.
3. Suspend partners that should lose student browsing access.
4. Remember: students must opt into partner-company visibility.

## Moderate Support, Legal, And Trust

1. Open support/legal queues.
2. Prioritize fake jobs, money requests, sensitive-document requests, impersonation, and privacy concerns.
3. Keep audit notes concise and factual.
4. Escalate production-account, payment, or legal-risk items to the owner.
