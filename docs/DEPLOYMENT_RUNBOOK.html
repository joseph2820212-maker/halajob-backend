Rules of the road

Each repo deploys independently. There is no shared build step and no monorepo tooling. The only cross-repo constraint is the backend contract: once a new backend version is live, the other three repos can be redeployed in any order.

Deploy backend first. When both a backend change and a client change are queued, ship the backend first. Clients tolerate a superset (unused new fields), but a client that expects new backend behaviour will break until the backend catches up.

halajob-backend → Node host

Prerequisites

  • Node.js ≥ 20 on the host (pinned in .nvmrc).
  • MongoDB reachable from the host — connection URL for CONNECTION_URL.
  • SMTP account for transactional mail — SMTP_* and HALAJOB_EMAIL_*.
  • Cron scheduler enabled by default (SCHEDULED_JOBS_ENABLED=true). Set to false if a separate worker will run cron.

Steps

  1. Clone the repo at the tagged commit.
  2. Copy .env.example → .env and fill in every non-optional variable (see the Environment section of the API reference).
  3. npm ci --ignore-scripts
  4. Run smoke: npm run smoke:import && npm run smoke:http
  5. Verify contract: npm run test:route-validation && npm run test:response-codes
  6. Start: npm start (behind your process manager, e.g. pm2 / systemd).
  7. Verify liveness: curl https://<host>/health → {"status": true, "message": "OK"}
  8. Verify CORS from a real client origin (should 204 on OPTIONS, 200 on GET).

Rolling deploy

  1. Bring up the new process on a spare port.
  2. Health-check the new port.
  3. Swap the reverse-proxy upstream.
  4. Drain and stop the old process.

Rollback

Redeploy the previous tagged commit. Database migrations are additive (Mongoose schemas accept unknown fields by default) — a rollback should not require a data-restore in normal releases.

halajob-mobile → Play Store / App Store

APK (tester)

Produced by the manual workflow_dispatch job in halajob-mobile/.github/workflows/ci.yml.

  1. Trigger Mobile CI → apk in GitHub Actions.
  2. Provide base_url (backend URL) and local_campus_auth = false for a normal tester build.
  3. Download halajob-mobile-apk artifact after the job completes.
  4. Also download halajob-mobile-apk-symbols and store it with the release — you will need it to symbolize any crash trace for this APK.

Play Store bundle

Local flow using the PowerShell scripts under halajob-mobile/scripts/:

# One-time signing setup (creates keystore/ + android/key.properties)
.\scripts\setup-android-signing.ps1

# Build a production AAB
.\scripts\build-android.ps1 -BuildTarget release-aab -BaseUrl https://jobzain.com

# Verify metadata before Play Console upload
.\scripts\assert-mobile-playstore-release.ps1 \
  -ArtifactName halajob-mobile-latest-release.aab \
  -ExpectedBaseUrl https://jobzain.com \
  -RequirePreflightSummary

Post-install check on device

  1. Sign in as a seeker; confirm the dashboard loads real data.
  2. Turn wifi off, force-close, reopen — cached dashboard should still show (no crash).
  3. Verify the app cannot reach a plain http:// URL (network_security_config).
