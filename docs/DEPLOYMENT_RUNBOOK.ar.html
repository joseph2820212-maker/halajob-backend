قواعد الطريق

كل مستودع يُنشَر باستقلال. لا خطوة بناء مشتركة ولا أدوات monorepo. القيد الوحيد بين المستودعات هو عقد الخادم: بمجرد أن تصبح نسخة جديدة من الخادم حية، يمكن إعادة نشر المستودعات الثلاثة الأخرى بأي ترتيب.

انشر الخادم أولاً. عندما يكون هناك تغيير خادم وتغيير عميل، اشحن الخادم أولاً. العملاء يتحملون superset (حقول جديدة غير مستخدمة)، لكن عميلاً يتوقع سلوك خادم جديد سيكسر حتى يلحق الخادم.

halajob-backend → مضيف Node

المتطلبات

  • Node.js ≥ 20 على المضيف (مثبّت في .nvmrc).
  • MongoDB قابل للوصول من المضيف — سلسلة الاتصال لـ CONNECTION_URL.
  • حساب SMTP للبريد المعاملاتي — SMTP_* وHALAJOB_EMAIL_*.
  • مجدول المهام مفعل افتراضياً (SCHEDULED_JOBS_ENABLED=true).

الخطوات

  1. استنسخ المستودع على الـ commit الموسوم.
  2. انسخ .env.example → .env واملأ كل المتغيرات غير الاختيارية.
  3. npm ci --ignore-scripts
  4. شغّل الدخان: npm run smoke:import && npm run smoke:http
  5. تحقق من العقد: npm run test:route-validation && npm run test:response-codes
  6. ابدأ: npm start (خلف مدير عمليات، مثل pm2 / systemd).
  7. تحقق من الحيوية: curl https://<host>/health → {"status": true, "message": "OK"}
  8. تحقق من CORS من عميل حقيقي.

نشر متجدد

  1. ارفع العملية الجديدة على منفذ احتياطي.
  2. افحص حيوية المنفذ الجديد.
  3. بدّل upstream في البروكسي العكسي.
  4. أوقف العملية القديمة.

التراجع

أعد نشر الـ commit السابق الموسوم. ترحيلات القاعدة إضافية — لا يجب أن يتطلب التراجع استعادة بيانات في الإصدارات العادية.

halajob-mobile → Play / App Store

APK (للاختبار)

يُنتج من مهمة workflow_dispatch اليدوية.

  1. شغّل Mobile CI → apk في GitHub Actions.
  2. وفّر base_url وlocal_campus_auth = false للبناء العادي.
  3. نزّل أثر halajob-mobile-apk.
  4. نزّل أيضاً halajob-mobile-apk-symbols واحفظه مع الإصدار — تحتاجه لرمز أي تعطل.

حزمة Play Store

تدفق محلي باستخدام سكربتات PowerShell:

# إعداد التوقيع مرة واحدة
.\scripts\setup-android-signing.ps1

# بناء AAB إنتاجي
.\scripts\build-android.ps1 -BuildTarget release-aab -BaseUrl https://jobzain.com

# تحقق من البيانات الوصفية قبل رفع Play Console
.\scripts\assert-mobile-playstore-release.ps1 \
  -ArtifactName halajob-mobile-latest-release.aab \
  -ExpectedBaseUrl https://jobzain.com \
  -RequirePreflightSummary

فحص ما بعد التثبيت

  1. سجّل دخول كباحث؛ تأكد أن اللوحة تعرض بيانات حقيقية.
  2. أطفئ Wi-Fi، أغلق التطبيق قسراً، أعد فتحه — يجب أن تعرض اللوحة المخبأة.
  3. تحقق أن التطبيق لا يستطيع الوصول إلى http:// عادي.
