Conventions

ID scheme
MongoDB ObjectId (24-char hex). Cross-references stored as ObjectId, not string.
Timestamps
Every schema opts into { timestamps: true } — createdAt / updatedAt are auto-managed by Mongoose.
Localized fields
Stored as embedded documents: { en: '…', ar: '…' }. Localization helpers pick the requested language and fall back to English.
Soft delete
Most user-facing collections use an isArchived / status flag rather than hard deleteOne.
Audit trail
audit_logs collection is append-only. TTL expiresAt per category (AUTH 2y, DATA/ADMIN 3y, PHI 7y, SYSTEM 1y).
Retention
data-retention jobs sweep old anonymised records nightly (scheduled:data-retention-cleanup in package.json).

Anchor collections

The three anchors most other collections point at:

  • employees — seeker/campus student accounts. Referenced by jobs (applicants), employee_cvs, career_passports, saved-jobs collections, notifications, and audit logs.
  • companies — employer accounts. Referenced by jobs, company_members (team seats), company_subscriptions, company_invoices, and interviews.
  • jobs — job postings. Referenced by job_employee_matches, job_invitations, job_reports, interviews, and every application-flow collection.

Foreign-key style references are enforced at the Mongoose layer (ref in schema plus optional populate). MongoDB does not enforce them at write time — the model-integrity verifier (npm run test:model-integrity in CI) is the safety net that catches dangling references.
