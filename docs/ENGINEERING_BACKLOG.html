How to use this backlog

This is the operational ticket register the incoming four-engineer team consumes day-one. Every ticket has an ID (stable across future updates), a priority tier, an owner role, a phase, an evidence pointer (file:line), a one-sentence fix shape, and an engineer-hour estimate.

Import into JIRA / Linear / GitHub Projects: the priority tag maps directly to a label; the owner tag maps to a team; the phase tag maps to a sprint or milestone. Effort is a coarse T-shirt estimate; treat it as a planning number, not a commitment.
