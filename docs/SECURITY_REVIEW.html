Scope

Cross-repo security posture at handoff. Covers the backend, the two JS frontends, and the mobile app. Threat model at the end covers the classes of attacks the current controls address and where residual risk still lives.

Everything here is running today. This is not a wish-list. The evidence is in app.js, vercel.json, AndroidManifest.xml, and the four repos' CI workflows. Anything missing is called out under residual risk.

Sign-off gates

Public launch is gated on these owner actions (they cannot be completed by an engineer):

  1. Rotate production secrets (JWT_SECRET, SMTP creds, provider keys) and archive the previous set.
  2. Audit live-database admin users; confirm each account matches an approved operator (script: npm run security:audit-users).
  3. Confirm HTTPS + HSTS chain on all four public origins.
  4. Confirm CI status green across all four repos.
