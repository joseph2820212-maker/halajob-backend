# Legacy planning docs

These documents were carried over from the `halajobe` monorepo when this
backend repo was split out. They cover product-wide planning (launch,
UX/navigation, brand, mobile handoff, and the repo-split runbook itself) that
predates the split and is not specific to the backend API.

They are kept for historical reference. New backend documentation should live
alongside its subject in `docs/` at the appropriate topic file, not here.

Product-wide planning that stays relevant across the four repos belongs in the
monorepo (`joseph2820212-maker/halajobe`) or should be re-homed to whichever
client repo it concerns.
