Introduction

Hala Job is a Syria-first hiring platform. This handoff covers the four production repositories, their contracts, and the operational envelope required to run them.

The platform ships today as four independent, redeployable applications plus one shared MongoDB. Everything else — mail, push, storage, AI — is pluggable and either disabled by default or gated behind an owner decision.

Product status

  • Backend surfaces 4,004 mounted routes across 15 modules — all with helmet, CORS allow-list, per-prefix rate limits, hpp, mongo-sanitize, and per-file upload guards.
  • Three JS frontends type-check strictly and ship with a hardened CSP (customer + admin no unsafe-eval; inline scripts pinned by sha256).
  • Mobile app blocks cleartext at the OS level, ships obfuscated APKs with a private symbols archive for post-hoc de-obfuscation.
  • All four repos have GitHub Actions CI with dependency audit steps (npm audit --omit=dev --audit-level=high — 0 vulnerabilities today).

Launch blockers requiring owner input

Code is production-hardened. Public launch still needs owner-side action on the following (documented in more detail in the Owner Handover Checklist):

  1. Rotate production secrets and re-audit privileged users in the live database.
  2. Run a live production smoke against the real deployment.
  3. Real accounts for SMTP, Firebase/push, Cloudinary or equivalent storage, AI provider (optional), domain/HTTPS, hosting, and backup/restore.
  4. Decide payments posture: accept manual/admin subscriptions for launch, or select an online payment provider.
  5. Decide Android signing/package/update strategy before public APK distribution.
