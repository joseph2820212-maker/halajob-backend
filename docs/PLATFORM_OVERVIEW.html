Four repositories, one product. The website, admin console, and mobile app all consume the same Express backend over HTTPS. There is no service-to-service traffic — the backend is the only stateful tier.

On-call runbook

First-response playbook for the common incidents. Every symptom starts as a client-side symptom — always confirm the backend is reachable before assuming a client bug.

SymptomLikely causeFirst checksEscalate
Website shows API offlineCORS or backend downcurl https://jobzain.com/health; check CORS_ORIGINSBackend on-call
Admin locked out for everyoneJWT_SECRET rotated or DB migrationLogin /dash/v1/auth/login from curl; check Mongo replica healthBackend on-call + owner
Mobile can't reach API on cellularCleartext or TLS chain issueConfirm server presents valid cert; verify only HTTPS URL is compiled inBackend + mobile owner
APK crashes with Dart stack in Play ConsoleObfuscated symbolsFetch matching halajob-mobile-apk-symbols artifact for the build SHA; feed to flutter symbolizeMobile owner
Legal reports queue frozenAdmin role missing permCheck GET /dash/v1/roles; run check:permissionsBackend + product owner
File uploads fail with 429Upload rate limiterConfirm from response headers; tune uploadLimiter if design intent has changedBackend on-call
Mail not deliveredSMTP creds or mailboxSend from same env with swaks; check spamOps owner
Cron jobs silentScheduler flag offSCHEDULED_JOBS_ENABLED, worker logs, DB countersBackend on-call
